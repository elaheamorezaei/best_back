# API Documentation — صفحه اخذ نمایندگی (`/franchise`)

> Base URL: `/api/v1`  
> Content-Type: `application/json`  
> این endpoint نیاز به احراز هویت ندارد.

---

## `POST /franchise/apply/`

ثبت درخواست اخذ نمایندگی. هنگامی که کاربر فرم صفحه `/franchise` را پر و ارسال می‌کند، فرانت‌اند این endpoint را صدا می‌زند.

---

### Request

```http
POST /api/v1/franchise/apply/
Content-Type: application/json
```

```json
{
  "fullName": "علی محمدی",
  "phone": "09123456789",
  "email": "ali@example.com",
  "city": "اصفهان",
  "province": "اصفهان",
  "franchiseType": "store",
  "investmentRange": "50m_150m",
  "hasSalesExperience": true,
  "description": "توضیحات اختیاری متقاضی"
}
```

### فیلدهای Request

| فیلد                 | نوع       | اجباری | توضیح                                         |
| -------------------- | --------- | :----: | --------------------------------------------- |
| `fullName`           | `string`  |   ✅   | نام و نام خانوادگی متقاضی                     |
| `phone`              | `string`  |   ✅   | شماره موبایل — فرمت: `09xxxxxxxxx`            |
| `email`              | `string`  |   ❌   | آدرس ایمیل (اختیاری، ممکن است ارسال نشود)     |
| `city`               | `string`  |   ✅   | نام شهر                                       |
| `province`           | `string`  |   ✅   | نام استان                                     |
| `franchiseType`      | `enum`    |   ✅   | نوع نمایندگی — مقادیر مجاز: جدول زیر          |
| `investmentRange`    | `enum`    |   ✅   | بازه سرمایه‌گذاری — مقادیر مجاز: جدول زیر     |
| `hasSalesExperience` | `boolean` |   ✅   | آیا متقاضی سابقه فروش دارد؟                   |
| `description`        | `string`  |   ❌   | توضیحات تکمیلی (اختیاری، ممکن است ارسال نشود) |

---

### مقادیر مجاز `franchiseType`

| مقدار    | برچسب نمایشی            |
| -------- | ----------------------- |
| `store`  | فروشگاهی (حضوری)        |
| `online` | اینترنتی (آنلاین)       |
| `hybrid` | ترکیبی (حضوری + آنلاین) |

### مقادیر مجاز `investmentRange`

| مقدار        | برچسب نمایشی            |
| ------------ | ----------------------- |
| `under_50m`  | کمتر از ۵۰ میلیون تومان |
| `50m_150m`   | ۵۰ تا ۱۵۰ میلیون تومان  |
| `150m_300m`  | ۱۵۰ تا ۳۰۰ میلیون تومان |
| `above_300m` | بیش از ۳۰۰ میلیون تومان |

---

### Response — موفق `201 Created`

```json
{
  "success": true,
  "message": "درخواست نمایندگی با موفقیت ثبت شد",
  "data": {
    "trackingCode": "FRN-2025-00483",
    "createdAt": "2025-07-01T14:22:00Z"
  }
}
```

| فیلد                | نوع      | توضیح                                   |
| ------------------- | -------- | --------------------------------------- |
| `data.trackingCode` | `string` | کد پیگیری که به کاربر نمایش داده می‌شود |
| `data.createdAt`    | `string` | تاریخ ثبت (ISO 8601)                    |

> فرانت‌اند `trackingCode` را بلافاصله بعد از ارسال موفق روی صفحه به کاربر نشان می‌دهد.

---

### Response — خطای اعتبارسنجی `422 Unprocessable Entity`

```json
{
  "success": false,
  "message": "شماره تماس معتبر نیست"
}
```

### Response — خطای سرور `500`

```json
{
  "success": false,
  "message": "خطا در ثبت درخواست. لطفاً دوباره تلاش کنید"
}
```

> **مهم:** مقدار `message` مستقیماً به کاربر نشان داده می‌شود. پیام را **فارسی و قابل‌فهم** بنویسید.

---

### نکات پیاده‌سازی

- اعتبارسنجی `phone` سمت سرور: باید با `09` شروع شده و ۱۱ رقم باشد.
- `email` اختیاری است — ممکن است اصلاً در body نباشد (نه `null`، نه `""`).
- `description` اختیاری است — همین حالت.
- `hasSalesExperience` همیشه `boolean` است (`true` یا `false`).
- درخواست‌های تکراری (همان `phone` در یک بازه زمانی) بهتر است rate-limit شوند.
